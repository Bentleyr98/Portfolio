If I get time, I would love to create more detailed notes by demonstrating each one on a web page.

  • Comment to explain why the code works, not how.
  • Remember that Symbols are a data type as well.
  • Using the keyword "delete" only removes the value, not the item.
  • Sets are a data structure that only holds unique values.
  • Remember that booleans can be powerful in logic. Look out for what values are inheritly false and true.
  • Ternary operators are a shorthand for an if/else statement. condition ? true : false. Could be very handy to use in code.
  • Always break in switches. And don't forget about the default condition.
  • Anything over a certain value is given the value/error of "infinity".
  • Keep functions "DRY".
  • Keep track of scope, OOP prinicples can be very handy in utilizing functions.
  • Callbacks are functions that are passed as an argument to a different function. It's like when an actress gets a "callback" to be heard from again. Our functions are doing the same and get the chance to act again.
